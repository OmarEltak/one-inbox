#### Facebook OAuth Flow (code path)
1. User clicks "Connect with Facebook" → hits `GET /connections/facebook/redirect`
2. `ConnectionController::facebookRedirect()` → `FacebookPlatform::getConnectUrl()` builds FB OAuth URL with scopes + redirect_uri
3. Browser redirects to `facebook.com/dialog/oauth?...`
4. User grants permissions → FB redirects to `GET /connections/facebook/callback?code=XXX`
5. `ConnectionController::facebookCallback()` → `FacebookPlatform::handleCallback()`:
   - Exchange code for short-lived token
   - Exchange for long-lived token (~60 days)
   - Fetch user profile (`/me`)
   - Create/update `ConnectedAccount`
   - `fetchPages()` → stores each FB Page with permanent page access token
   - `subscribePage()` → subscribes to webhook events
   - `detectInstagramAccount()` → checks for linked IG business account
6. Redirect back to Connections page with success message

#### Telegram Connection Flow (code path)
1. User enters bot token in modal → `POST /connections/telegram/connect`
2. `ConnectionController::telegramConnect()` → `TelegramPlatform::handleCallback()`:
   - Validates token via `getMe` API
   - Creates `ConnectedAccount` + `Page` for the bot
   - `setWebhook()` → registers `{APP_URL}/api/webhooks/telegram` with Telegram

#### Incoming Message Flow (all platforms)
1. Platform POSTs to webhook URL → Controller logs to `webhook_logs` table → dispatches `ProcessIncomingMessage` job
2. Job routes to platform-specific processor (Meta Messenger, WhatsApp, Telegram)
3. Creates/finds Contact + ContactPlatform
4. Creates/finds Conversation
5. Creates Message record
6. Dispatches `ScoreLeadJob` (always)
7. If AI enabled → dispatches `SendAiResponse` with random delay

#### Outgoing Message Flow
1. User types in Inbox → `Inbox\Index::sendMessage()`
2. Creates local Message record
3. Dispatches `SendPlatformMessage` job
4. Job routes to correct platform API (Meta Graph API, Telegram Bot API)
