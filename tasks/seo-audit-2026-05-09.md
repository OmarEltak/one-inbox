### B2. CDN cannot cache marketing pages (CRITICAL)

```
Cache-Control: no-cache, private
Set-Cookie: XSRF-TOKEN=...; ot1-pro-session=...
cf-cache-status: DYNAMIC
```

Laravel's default web middleware stack runs `StartSession` on every request, which:
- Sets a session cookie (busts CDN cache).
- Sets `Cache-Control: no-cache, private` (forbids CDN cache).

Result: Every public marketing visit forces a PHP cold path through routes → middleware → Blade compile → DB connection. Cloudflare is just a proxy for these pages.

**Fix (recommended approach):**
1. Create a `CacheableResponse` middleware:
   ```php
   public function handle($request, $next) {
       $response = $next($request);
       if ($request->isMethod('GET') && !auth()->check()) {
           $response->headers->set('Cache-Control', 'public, max-age=300, s-maxage=3600');
           $response->headers->remove('Set-Cookie');  // strip session cookie
       }
       return $response;
   }
   ```
2. Apply only to public marketing routes (home, features, /vs/*, /industries/*, /blog, /blog/{slug}, /whatsapp-inbox, /instagram-dm, /facebook-messenger, /telegram-inbox, /about, /contact).
3. Or move marketing routes to a separate route group **without** the `web` middleware (pure Blade rendering, no session).
4. Add Cloudflare Page Rule: `ot1-pro.com/blog/*` → Cache Everything, Edge TTL 1 day.

Expected impact: **TTFB on cached visitors drops from 3–6s to <50ms**. Direct improvement to LCP, INP, and Google's perception of site quality. This single fix is worth more than any 10 on-page changes.

### B3. Slow original render even on origin (HIGH)
Even without CDN, 5+ second TTFB on simple Blade pages is wrong. Likely causes (one or more):
- Eager-loading too much in middleware
- DB connection on every request even when not needed
- Hetzner CX22/32 server in Germany while Cloudflare CF-RAY shows MRS (Marseille) — geographic latency is fine for EU/MENA but not the cause here.
- Missing OPcache or `php artisan config:cache` / `route:cache` / `view:cache` in production.

**Action:** SSH into prod and run:
```bash
php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan event:cache
```
Verify OPcache enabled in `php.ini` (`opcache.enable=1`, `opcache.validate_timestamps=0` in prod).

`Content-Encoding: br` confirmed. Good.

## Section D — Schema / Structured Data

### D1. ✅ Schema is broadly well-implemented
- `SoftwareApplication` on every page (informational, harmless duplicate)
- `FAQPage` on home, /whatsapp-inbox, /instagram-dm, /facebook-messenger, /telegram-inbox, /vs/* (all 5), /industries/* — great coverage
- `Article` + `BreadcrumbList` on blog posts — correct types

### D2. Article schema is missing required `image` (HIGH)
Sample blog post Article JSON-LD:
```json
{
  "@type": "Article",
  "headline": "WhatsApp CRM: The Complete Guide for 2026",
  "datePublished": "2026-05-05T20:06:36+00:00",
  ...
  // ❌ no "image" property
  // ❌ no "author.url" or author Person type
}
```

Google's Article rich-result requirements: `image`, `headline`, `datePublished`, `author`. **Without `image`, no rich result.** Currently every blog post is shipping invalid Article schema.

**Fix:** in the Article schema partial, add:
```json
"image": ["https://ot1-pro.com/og-image.png"]
```
(use the post's hero image when you have one; fall back to the default OG.)

### D3. Breadcrumb has duplicate URLs (HIGH)
```json
{ "position": 2, "name": "Blog",              "item": "https://ot1-pro.com/blog" },
{ "position": 3, "name": "WhatsApp Business", "item": "https://ot1-pro.com/blog" }  // ❌ same URL
```
Google's structured-data validator will flag this. Either:
- Build category/cluster URLs (`/blog/category/whatsapp`) and link position 3 to those, OR
- Remove position 3 entirely (3-level breadcrumb: Home → Blog → Article).

### D4. Blog index missing Blog/CollectionPage schema (MEDIUM)
`/blog` only has `SoftwareApplication`. Add:
```json
{
  "@type": "Blog",
  "url": "https://ot1-pro.com/blog",
  "name": "One Inbox Blog",
  "blogPost": [ /* array of recent BlogPosting refs */ ]
}
```

### D5. Missing Organization schema on home (MEDIUM)
Add Organization schema (separate from SoftwareApplication) with:
- `logo`
- `url`
- `sameAs`: links to Facebook page, Instagram, LinkedIn, X/Twitter
- `contactPoint` for support email/WhatsApp number (+201026361218 from MEMORY.md)

This builds entity recognition — Google starts treating "One Inbox" as a known brand.

### D6. SoftwareApplication has `price: 0` (LOW)
`offers.price: "0"` declares the product as free. Valid for the free tier, but consider an `AggregateOffer` with low/high price (`0` to `79`) once you ship Pricing.

### D7. No `aggregateRating` (LOW)
Add once you have ≥5 reviews — gets star ratings in SERP, big CTR boost.
